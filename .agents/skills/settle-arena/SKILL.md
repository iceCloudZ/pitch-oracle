# settle-arena

Settles finished matches end-to-end: scores → leaderboard → bet settlement → poster → upload. Designed for the pitch-oracle project (single-branch `main` workflow).

## Prerequisites

All commands run from the project root `D:\github\pitch-oracle`. The following must be true:
- `.env` exists with `FOOTBALL_DATA_API_KEY`, `DEEPSEEK_API_KEY`, `QWEN_API_KEY` set.
- Node 20+ and `npx tsx` available.
- Edge browser at `C:\Program Files (x86)\Microsoft\Edge\Application\msedge.exe` (for headless screenshot).
- SSH host `43` (43.156.98.94) reachable for poster upload.

## Steps

Execute all steps in order. Report results to the user after each major step.

### 1. Fetch scores from football-data

```bash
npx tsx -e "(async()=>{const m=await import('./src/data/fixtures-results.js');const r=await m.fetchFixturesResultsWithTeams({apiKey:process.env.FOOTBALL_DATA_API_KEY,competition:'WC'});console.log(r.length+' finished matches');r.forEach(x=>console.log('  '+x.homeTeam+' '+x.homeScore+'-'+x.awayScore+' '+x.awayTeam+' (fd:'+x.matchId+')'))})().catch(e=>console.error(e.message))"
```

Set env vars before running. On Windows cmd:
```cmd
set "FOOTBALL_DATA_API_KEY=<value>" && set "DEEPSEEK_API_KEY=<value>" && set "QWEN_API_KEY=<value>" && npx tsx ...
```

Identify which of the returned matches correspond to sporttery matchIds in `data/match-index.json`. Report the scores found.

### 2. Run the score pass

```cmd
set "FOOTBALL_DATA_API_KEY=<value>" && set "DEEPSEEK_API_KEY=<value>" && set "QWEN_API_KEY=<value>" && npx tsx src/cli.ts score
```

This joins football-data results onto sporttery matchIds via `data/match-index.json`, scores all agent predictions, settles Kelly bets, and writes `data/leaderboard.json`.

Check the `[join]` warnings in output. If a match is skipped with `no fixture found`, the sporttery matchId may be missing from `data/match-index.json` — add it manually (matchId → {homeTeam, awayTeam} in Chinese). If `unmapped team name`, add the missing English alias to `src/data/team-names.ts` and re-run.

### 3. Settle bet slips in data/my-bets.json

Read `data/my-bets.json`. For each slip with `status: pending`:
- Look up the actual score for each match in the slip.
- For `hhad` (让球) bets: apply the goalLine to the actual score to determine 让胜/让平/让负.
  - goalLine < 0 means the home team concedes (e.g. goalLine -1 means home must win by 2+ for 让胜).
  - goalLine > 0 means the home team receives (e.g. goalLine +2 means home can lose by 1 for 让平).
- For `had` (胜平负) bets: compare directly with the full-time result.
- For each multi-leg (串关) entry: ALL legs must win for the entry to pay out. Combined odds = product of individual odds. Payout = combinedOdds × multiplier × 2.
- Update `status` to `won`/`lost`, fill `pnl` (payout - stake for won, -stake for lost), and the slip `summary`.

Report the settlement to the user in a clear table.

### 4. Screenshot the poster

The poster HTML is `poster.html`. If the leaderboard data has changed significantly, update the HTML first (edit the leaderboard table, bet slip rows, and cumulative PnL). Then screenshot:

```cmd
"C:\Program Files (x86)\Microsoft\Edge\Application\msedge.exe" --headless --disable-gpu --window-size=800,1100 --screenshot="D:\github\pitch-oracle\poster.png" "file:///D:/github/pitch-oracle/poster.html"
```

### 5. Upload to icebao.top

To bypass CDN cache, increment the poster filename (poster2.png, poster3.png, ...):

```cmd
scp poster<n>.png 43:/tmp/poster<n>.png
ssh 43 "sudo cp /tmp/poster<n>.png /opt/baoxian/web/poster<n>.png && sudo chmod 644 /opt/baoxian/web/poster<n>.png"
```

Verify public access:
```cmd
curl -s -o nul -w "HTTP %{http_code} %{size_download} bytes" -L --max-time 15 "https://icebao.top/poster<n>.png"
```

Report the public URL to the user.

### 6. Commit to main (single branch)

```cmd
git add -A && git commit -m "data: settle <date> matches + leaderboard + poster" && git push origin main
```

## Key notes

- football-data.org's FINISHED endpoint is flaky (sometimes omits matches). The results persistence (`data/results.json`) mitigates this — confirmed scores accumulate and survive re-runs.
- sporttery delists finished matches, so `data/match-index.json` is the only source of matchId↔team-name mapping for scoring. Always ensure new predicted matches are in the index.
- GLM predictions (`glm-blind`/`glm-odds`) are manually authored (type `manual` in agents.json). They are NOT auto-generated by `predict`.
- me-ai predictions reflect the user's REAL bet-slip judgments, not DeepSeek's auto-generated picks. They were rewritten to match actual bets.
